OWASP **Juice Shop** is a deliberately insecure, modern web application created by the OWASP Juice Shop project to help people learn and practice web application security.

It’s designed to look and feel like a real e-commerce site that sells juices, but it intentionally includes a wide range of vulnerabilities—from beginner-friendly issues to advanced flaws—so learners can safely explore how attacks work and how to defend against them. The app is commonly used for:

* **Hands-on training** for developers, AppSec engineers, and security testers
* **Workshops and CTFs** (capture-the-flag challenges) with guided or self-paced tasks
* **Tool practice** (SAST/DAST, scanners, proxies like OWASP ZAP, etc.) in a safe environment
* **Secure coding education** by demonstrating what “bad” looks like in realistic flows (login, search, checkout, profiles, API calls, admin features)

Juice Shop typically covers vulnerability categories such as broken access control, injection, XSS, authentication/session issues, security misconfiguration, sensitive data exposure, API weaknesses, and logic flaws—mapping well to the OWASP Top 10 mindset and modern web app patterns (single-page app behavior, APIs, client-side heavy flows).

To access Juice shop you need to access http://localhost:3001. You need to signup for the application and then login and use the rest of the application as an authenticated user
